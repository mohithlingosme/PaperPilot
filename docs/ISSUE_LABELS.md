# Issue Labels

Create the following labels in GitHub UI for issue management:

## Priority
- **P0** (color: #d73a49, description: Critical - fix immediately)
- **P1** (color: #f66a0a, description: High priority - fix soon)
- **P2** (color: #fbca04, description: Medium priority - fix when possible)

## Area
- **backend** (color: #0e8a16, description: Backend/API related issues)
- **frontend** (color: #1d76db, description: Frontend/web related issues)
- **infra** (color: #5319e7, description: Infrastructure/deployment issues)
- **docs** (color: #006b75, description: Documentation issues)
- **security** (color: #b60205, description: Security related issues)
- **ai** (color: #e99695, description: AI/ML related issues)
- **data** (color: #d4c5f9, description: Data processing/storage issues)

## Type
- **bug** (color: #d73a49, description: Something isn't working)
- **feature** (color: #a2eeef, description: New feature or request)
- **chore** (color: #fef2c0, description: Maintenance or cleanup)
- **refactor** (color: #fbca04, description: Code refactoring)

## Status
- **blocked** (color: #d93f0b, description: Issue is blocked by another issue)
- **needs-review** (color: #fbca04, description: Needs code review or feedback)
